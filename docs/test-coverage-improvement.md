## 3. Highest-Value Coverage Gaps

### Gap A â€” `MangaDexCallExecutor` concurrency, interrupts, and propagation

- **Location/files:** `backend/src/main/java/com/mangatrack/manga/MangaDexCallExecutor.java`
- **Why it matters:** This is the resilience boundary for every outbound MangaDex call. A leaked semaphore permit (failure to release in `finally`) would permanently shrink outbound concurrency and eventually deadlock check runs. Incorrect interrupt handling would swallow shutdown signals. These are classic, hard-to-spot concurrency bugs and the class was just extracted.
- **Existing tests:** None directly. `MangaDexServiceTest` covers retry/backoff *through HTTP*, but always at the service layer (which swallows exceptions to empty) and never asserts permit release, interrupt behavior, or `maxAttempts == 1`.
