## 3. Highest-Value Coverage Gaps

### Gap D — `MangaCheckerService.computeNextCheckDate` scheduling branches

- **Location/files:** `backend/src/main/java/com/mangatrack/manga/MangaCheckerService.java`, `backend/src/test/java/com/mangatrack/manga/MangaCheckerServiceTest.java`
- **Why it matters:** When a check finds *no new chapter*, this method decides when the manga is re-checked. It has three branches: no known update day → tomorrow; tomorrow *is* the update day → tomorrow; otherwise → the next occurrence of the update day. A regression here would silently re-check titles on the wrong day (too early = wasted MangaDex calls; too late = delayed chapter alerts). This is pure date arithmetic and the natural place for off-by-one / `TemporalAdjusters.next` bugs.
- **Existing tests:** `sameChapter_savesOnceAndDoesNotPublish` reaches this path and sets `updateDayOfWeek = MONDAY`, but **never asserts the resulting `nextCheckDate`**, so all three branches are unverified.
- **Missing cases:** `updateDayOfWeek == null` → today + 1; `updateDayOfWeek` equals tomorrow's day → today + 1 (tomorrow); `updateDayOfWeek` is some other day → `today.with(next(updateDay))`.
- **Suggested tests:** extend `MangaCheckerServiceTest` with three same-chapter scenarios asserting `nextCheckDate`, computed relative to `LocalDate.now()` (matching the suite's existing `plusDays(...)` convention) so they stay deterministic.
- **Risk level:** Low (test-only, no Spring context).
- **Suggested validation:** `./mvnw test -Dtest=MangaCheckerServiceTest`
- **Status:** Implemented

### Gap E — `MangaCheckerService` cover-art backfill (`fetchCoverUrl`)

- **Location/files:** `backend/src/main/java/com/mangatrack/manga/MangaCheckerService.java`, `backend/src/test/java/com/mangatrack/manga/MangaCheckerServiceTest.java`
- **Why it matters:** When a manga already has a MangaDex id but no cover URL (e.g. resolved by title where the search result had no cover, or pre-existing rows), the checker backfills the cover via `fetchCoverUrl` before fetching chapters. The cover drives a user-visible thumbnail. None of the existing tests leave `coverUrl == null` with a known id, so this branch — and both of its sub-paths (cover found → set; cover absent → stays null) — is never reached.
- **Existing tests:** Every checker test either sets `coverUrl` up front or resolves it from the search result, so `fetchCoverUrl` is never invoked.
- **Missing cases:** known id + null cover + `fetchCoverUrl` returns a URL → cover set; known id + null cover + `fetchCoverUrl` empty → cover stays null (no crash).
- **Suggested tests:** extend `MangaCheckerServiceTest` with two scenarios where `coverUrl` starts null and a `mangadexId` is present.
- **Risk level:** Low.
- **Suggested validation:** `./mvnw test -Dtest=MangaCheckerServiceTest`
- **Status:** Implemented

### Gap F — `MangaController.add` duplicate-title conflict (409)

- **Location/files:** `backend/src/main/java/com/mangatrack/manga/MangaController.java`, `backend/src/test/java/com/mangatrack/manga/MangaControllerTest.java`
- **Why it matters:** `add` wraps the save in a `try/catch (DataIntegrityViolationException)` that translates the unique-title constraint violation into a `409 CONFLICT` with an `Already tracking "<title>"` message. This is the user-visible duplicate-prevention behavior and a distinct error branch. It is currently untested — existing `add` tests cover only validation 400s and the 201 happy path. A regression (swallowing the exception, wrong status, or auto-subscribing after a failed save) would go unnoticed.
- **Existing tests:** `add_blankTitle...`, `add_missingTitle...`, `add_validTitle...` — no conflict path.
- **Missing cases:** `repository.save` throws `DataIntegrityViolationException` → 409 with `Conflict` envelope and the title in the message; `subscriptionService.autoSubscribeDefaultUser` is **not** called when the save fails.
- **Suggested tests:** extend `MangaControllerTest` with a conflict scenario (stub `save` to throw), asserting status, `$.error`, `$.message`, and no auto-subscribe.
- **Risk level:** Low (slice test, no behavior change).
- **Suggested validation:** `./mvnw test -Dtest=MangaControllerTest`
- **Status:** Implemented
